    Introduction
    

    


    We ask you for the good progress of this evaluation to
 
    respect the following rules:
 
    

    - Be courteous, polite, respectful and constructive in all
 
      situations during this exchange. The bond of trust between the
 
      community 42 and you depends on it.

    

    - Highlight to the person (or group) assessed the
 
      possible malfunctions of work done, and take the time
 
      to discuss and discuss.

    

    - Accept that there may sometimes be differences
 
      of interpretation on the subject's requests or the scope of the
 
      features. Stay open-minded about 
      the other's vision
 (is he or she right or wrong?), And write down the most

      honestly possible. The pedagogy of 42 only makes sense if
 
      the peer-evaluation is done seriously.

    

    

    

    Guidelines
    

    


    - You should only evaluate what is on the student / group's rendering GiT repository
 
      .

    

    - Make sure that the GiT repository is the one
 
      corresponding to the student or group and the project.

    

    - Meticulously verify that no malicious alias has been used
 
      to mislead you and have you evaluate anything other than
 
      the content of the official repository.

    

    - Any script deemed to facilitate the evaluation provided by one of the two
 
      parties must be rigorously checked by the other party to
 
      avoid unpleasant surprises.

    

    - If the corrective student has not yet done this
 
      project, it is mandatory for this student to read the

      subject in full before starting this defense.

    

    - Use the flags available on this scale to signal an
 
      empty rendering, non-functional, a standard fault if applicable, a
 
      cheating case, etc. In this case, the evaluation is completed and the
 
      final grade is 0 (or -42 in the special case of
 
      cheating). However, except cheating, you are encouraged to
 
      continue to exchange around the work done (or not done
 
      precisely) to identify the problems that led to this
 
      situation and avoid them for the next rendering.

    

    - As a proofreader be strict on the notation of the explanations
 
      that the student is supposed to provide you on the functioning of the

      artificial neural networks. The concepts, although complex in
 
      application, are simple in theory, you must have understood
 
      all of its explanations to give the maximum number of points.

    Implementation
    

    This section is the evaluation of the implementation of the program.

    

    

    modularity
    

    
Does the artificial neural network contain at least 2 hidden layers?

    (so that the network is at least 4 deep by counting the input and output layers)

    

    

    

    

    





    

    

    Separation of the dataset
    

    
Is the dataset split into two, with one part for training
 
    and another part for validation? (The validation set allows
 
    evaluation of the model's performance during the training phase on
 
    data that are unknown to it).

    

    

    

    

    





    

    

    Implementing the softmax function
    

    
Is the softmax function implemented on the output layer of
 
    the neural network? This point is important because the softmax allows to have
 
    an output in the form of probabilistic distribution.

    

    

    

    

    





    

    

    

    

    

    

    Training
    

    This section is the evaluation of the training phase of the model. Run the program and note the following points.

    

    

    Viewing
    

    
Is something happening?

    - Is there a display of metrics at the end of each epoch? (similar to what is shown in the subject)
 
    - There is a graph display of learning curves at the end of the training?

      (there must be at least two curves, the training curve and the validation curve)

    

    

    

    

    





    

    

    metrics
    

    
Do we have at least one metric for evaluation on the 
    training dataset
 and one for validation? Are the 
    chosen metrics / metrics
 correct for binary classification? (If it's the
 
    mean square error, for example, root mean square error, for example binary cross-entropy loss
 
    is good, otherwise ask for additional justifications).

    

    

    

    

    





    

    

    Model
    

    
How is the training?

    - The model learns? (the value of the training and validation metrics
 
      improves overall as the training progresses)
 
    - We are not in a situation of overfitting.

    

    

    

    

    





    

    

    Save the template
    

    
Is the model saved at the end of the training?

    (by model I mean the network as a whole, that is, its topology and its weights)

    

    

    

    

    





    

    

    

    

    

    

    Prediction
    

    This section is the evaluation of the prediction phase of the model on the test dataset. Run the program and note the following points.

    

    

    Loading
    

    
The prediction program to work must somehow
 
    load the weights and topology of the model as well as the dataset on which the
 
    predictions will be made.

    - The trained model is correctly loaded (check that the values ​​are not hard in the code for example).

    - The test dataset is well loaded.

    

    

    

    

    





    

    

    metrics
    

    
The prediction program evaluates the model on the test dataset with the 
    binary cross entropy error function
 (if it is not the case the next question
 
    can not be evaluated).

    

    

    

    

    





    

    

    Model performance
    

    
Follow these instructions for this question:
 
    - Download the evaluation.py program from the subject resources.

    - Run it with the command 'python evaluation.py' (This program will download
 
       the dataset from the resources, mix it and split it into two parts).

    - Train a new model on the dataset named 'data_training.csv'.

    - Predict the dataset 'data_test.csv' with this model.

    

    You can redo up to 3 trainings and then evaluate the
 
    question on the best prediction performance obtained (indeed the training
 
    of a model depends on certain random factors such as the initialization of

    weight, several trainings will not converge towards the same performance).

    

    Look at the value of the prediction metric on the test dataset,
 
    - The value is above 0.35 => 0
 
    - The value is between 0.35 and 0.25 => 1
 
    - The value is between 0.25 and 0.18 => 2
 
    - The value is between 0.18 and 0.13 => 3
 
    - The value is between 0.13 and 0.08 => 4
 
    - The value is below 0.08 => 5

    

    

    

    

    Rate it from 0 (failed) through 5 (excellent)
